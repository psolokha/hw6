# Критерии приёмки: Анна — турист (US-A1 … US-A10)

Каждый критерий оформлен так:

- **Given** — начальное условие  
- **When** — действие пользователя  
- **Then** — проверяемый результат  

---

## US-A1

**История:** Как **турист**, я хочу **найти город или область поиска через строку поиска**, чтобы **увидеть достопримечательности в нужном месте**.

- 
**Given**  открыт экран выбора локации и доступен поиск по API карт.  
**When**  пользователь вводит допустимое название населённого пункта и подтверждает выбор.  
**Then**  приложение привязывает область поиска к выбранному результату и показывает каталог достопримечательностей для этой области (или сообщение об отсутствии результатов поиска).

- 
**Given**  пользователь ввёл строку без подходящих подсказок.  
**When**  он завершает поиск.  
**Then**  отображается понятное состояние «ничего не найдено» с возможностью изменить запрос.

---

## US-A2

**История:** Как **турист**, я хочу **фильтровать список по типу/категориям** (в рамках API), чтобы **сузить подборку под свой день**.

- 
**Given**  отображается каталог для выбранной локации и API возвращает категории.  
**When**  пользователь выбирает одну или несколько доступных категорий.  
**Then**  список обновляется и содержит только объекты, соответствующие выбранным фильтрам (или пустой список с пояснением).

- 
**Given**  активны фильтры.  
**When**  пользователь сбрасывает фильтры.  
**Then**  список обновляется и показывает все объекты текущей локации без ограничения по категориям на той же странице пагинации (при пагинации — с возвратом на первую страницу).

---

## US-A3

**История:** Как **турист**, я хочу **открывать карточки объектов** с описанием и координатами, чтобы **решить, интересен ли мне маршрут по этому району**.

- 
**Given**  в списке есть объект.  
**When**  пользователь открывает карточку.  
**Then**  отображаются как минимум название и координаты; описание и медиа показываются, если они есть в ответе API.

- 
**Given**  открыта карточка объекта из каталога.  
**When**  пользователь нажимает «Назад».  
**Then**  отображается предыдущий экран списка с той же выбранной локацией и теми же активными фильтрами, что и до открытия карточки.

---

## US-A4

**История:** Как **турист**, я хочу **задать ориентир длины в км и старт от текущей геолокации**, чтобы **получить пеший маршрут без ручного расчёта дистанции**.

- 
**Given**  разрешена геолокация и координаты получены.  
**When**  пользователь на экране построения маршрута вводит значение в допустимом диапазоне 2–50 км и подтверждает старт «текущее местоположение».  
**Then**  приложение использует полученные координаты как старт кольца и принимает ориентир длины для расчёта вариантов.

- 
**Given**  пользователь ввёл длину вне диапазона 2–50 км.  
**When**  он пытается продолжить.  
**Then**  отображается валидация с указанием допустимых границ без запуска построения маршрута.

---

## US-A5

**История:** Как **турист**, я хочу **получить до трёх вариантов кольцевого маршрута с нумерацией порядка и итоговой длиной**, чтобы **выбрать прогулку «как раз» по силам**.

- 
**Given**  для выбранной области и старта в выборке не менее трёх достопримечательностей.  
**When**  пользователь успешно запускает построение.  
**Then**  отображается от одного до трёх вариантов; у каждого варианта указаны суммарная длина в километрах или метрах и порядок посещения достопримечательностей 1…N, где N ≥ 3; маршрут кольцевой: старт и финиш совпадают, суммарная длина включает замыкающий отрезок к старту.

- 
**Given**  после отбора объектов для маршрута доступно менее трёх достопримечательностей.  
**When**  пользователь запускает построение.  
**Then**  успешный список вариантов маршрута не показывается; отображается сообщение о том, что для маршрута нужно не менее трёх точек, с предложением расширить область или ослабить фильтры.

---

## US-A6

**История:** Как **турист**, я хочу **сохранить выбранный маршрут в избранное**, чтобы **вернуться к нему в течение поездки**.

- 
**Given**  пользователь просматривает экран с выбранным вариантом маршрута.  
**When**  он нажимает «Сохранить в избранное».  
**Then**  в разделе избранного сохраняются суммарная длина варианта и упорядоченный список остановок с координатами; запись отображается в списке избранного.

- 
**Given**  маршрут уже в избранном.  
**When**  пользователь открывает избранное.  
**Then**  сохранённый маршрут доступен для просмотра без повторного построения.

---

## US-A7

**История:** Как **турист**, я хочу **просматривать каталог без обязательного перехода к маршруту**, чтобы **спокойно изучить город в своём темпе**.

- 
**Given**  выбрана локация и загружен каталог.  
**When**  пользователь только листает список и открывает карточки.  
**Then**  приложение не требует запуска «Собрать маршрут» и не блокирует выход из сценария каталога.

- 
**Given**  пользователь на экране каталога.  
**When**  он сворачивает приложение и снова открывает его из списка недавних.  
**Then**  восстанавливается экран, на котором пользователь был до сворачивания (каталог или карточка из каталога); принудительного перехода на экран «Собрать маршрут» нет.

---

## US-A8

**История:** Как **турист**, я хочу **открыть достопримечательность во внешнем приложении карт или на встроенной карте**, чтобы **запомнить расположение на карте города**.

- 
**Given**  открыта карточка объекта с валидными координатами.  
**When**  пользователь нажимает «Открыть в картах».  
**Then**  запускается внешнее приложение карт с передачей широты и долготы из карточки через Intent с `geo:` на Android или через открытие карт по координатам на iOS.

- 
**Given**  реализация использует встроенный фрагмент карты вместо внешнего приложения.  
**When**  пользователь открывает карточку и переходит к карте.  
**Then**  на фрагменте отображается маркер в координатах объекта из API.

- 
**Given**  система не может открыть внешние карты (нет подходящего приложения или отклонён запрос ОС).  
**When**  пользователь нажимает «Открыть в картах».  
**Then**  отображается сообщение об ошибке и координаты объекта дублируются текстом с кнопкой «Скопировать координаты».

## US-A9

**История:** Как **турист**, я хочу **задать область через геолокацию «рядом со мной»**, чтобы **не вводить название города в спешке**.

- 
**Given**  пользователь выбрал режим «Рядом со мной».  
**When**  он выдал разрешение на геолокацию.  
**Then**  приложение получает координаты устройства и загружает каталог POI в круге с центром в этой точке и фиксированным радиусом поиска из настроек клиента (одно и то же значение для всех пользователей в сборке), без ввода названия города.

- 
**Given**  пользователь отказал в геолокации.  
**When**  он пытается использовать «Рядом со мной».  
**Then**  показывается объяснение и предлагается выбор города вручную или повторный запрос разрешения.

---

## US-A10

**История:** Как **турист**, я хочу **перейти от списка мест к построению маршрута за минимум шагов**, чтобы **начать прогулку без долгой настройки**.

- 
**Given**  отображается список достопримечательностей для текущей локации.  
**When**  пользователь нажимает «Собрать маршрут».  
**Then**  сразу открывается экран ввода ориентира длины и выбора старта; между списком и этим экраном нет обязательных промежуточных шагов, кроме системных запросов разрешений (геолокация).

- 
**Given**  локация уже задана.  
**When**  пользователь переходит к построению маршрута.  
**Then**  контекст области поиска переносится на экран маршрута без повторного ввода города.
