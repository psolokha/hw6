# Критерии приёмки: Елена — деловой путешественник (US-E1 … US-E8)

Каждый критерий оформлен так:

- **Given** — начальное условие  
- **When** — действие пользователя  
- **Then** — проверяемый результат  

---

## US-E1

**История:** Как **деловой путешественник**, я хочу **задать город поиском в один-два шага**, чтобы **не тратить время на сложный выбор области**.

- 
**Given**  открыт экран выбора локации.  
**When**  пользователь вводит название города в поиск и выбирает один результат из списка подсказок.  
**Then**  локация считается заданной и активен переход к каталогу; поля «район» и «радиус» не являются обязательными для этого перехода.

- 
**Given**  пользователь уже выбрал город.  
**When**  он повторно открывает экран выбора локации.  
**Then**  отображается ранее выбранное название города; смена города выполняется через тот же поиск и список подсказок без отдельного многошагового мастера.

## US-E2

**История:** Как **деловой путешественник**, я хочу **собрать кольцевой маршрут по ориентиру в км со стартом от геолокации или выбранной точки**, чтобы **вписать прогулку в свободный слот**.

- 
**Given**  задана область каталога.  
**When**  пользователь вводит ориентир длины в диапазоне 2–50 км и подтверждает старт (геолокация или точка на карте).  
**Then**  запускается расчёт кольцевого маршрута с использованием выбранного старта.

- 
**Given**  введено значение вне 2–50 км.  
**When**  пользователь подтверждает ввод.  
**Then**  показывается ошибка валидации; расчёт не стартует.

---

## US-E3

**История:** Как **деловой путешественник**, я хочу **видеть нумерацию порядка посещения и итоговую длину варианта**, чтобы **быстро сориентироваться без лишних экранов**.

- 
**Given**  расчёт маршрута успешен.  
**When**  пользователь открывает экран вариантов.  
**Then**  для каждого варианта на одном экране без раскрытия блоков видны нумерация остановок 1…N и итоговая длина (в километрах или метрах).

- 
**Given**  несколько вариантов.  
**When**  пользователь переключается между карточками вариантов.  
**Then**  нумерация и длина соответствуют выбранному варианту и не дублируют данные другого варианта.

---

## US-E4

**История:** Как **деловой путешественник**, я хочу **открыть маршрут или точку во внешнем приложении карт**, чтобы **идти привычным навигатором** (пошаговая навигация только во внешнем приложении карт; внутри нашего приложения голосовых подсказек «куда повернуть» нет).

- 
**Given**  отображается список остановок маршрута.  
**When**  пользователь нажимает «Открыть в картах» у конкретной остановки.  
**Then**  во внешнее приложение карт передаются широта и долгота этой остановки: на Android через `geo:`-Intent, на iOS через системную ссылку на карты с координатами.

- 
**Given**  отображается экран варианта маршрута.  
**When**  пользователь выбирает действие «Открыть в картах» для всего маршрута.  
**Then**  открывается внешнее приложение карт с координатами первой остановки выбранного варианта (остальные точки доступны по отдельным действиям на том же экране).

- 
**Given**  ОС не открывает внешнее приложение карт.  
**When**  пользователь нажимает «Открыть в картах».  
**Then**  показывается сообщение об ошибке; встроенная пошаговая навигация в приложении не отображается и не требуется.

## US-E5

**История:** Как **деловой путешественник**, я хочу **получить к карточке объекта быстрый путь «открыть в картах»**, чтобы **сразу выйти к навигации без сборки маршрута**.

- 
**Given**  открыта карточка достопримечательности.  
**When**  пользователь нажимает «Открыть в картах».  
**Then**  происходит переход во внешнее приложение карт без обязательного посещения экрана «Собрать маршрут».

- 
**Given**  карточка открыта из списка.  
**When**  пользователь возвращается назад.  
**Then**  он снова в каталоге; сценарий маршрута не запущен автоматически.

---

## US-E6

**История:** Как **деловой путешественник**, я хочу **видеть в карточке достаточно данных из API (название, координаты, описание при наличии)**, чтобы **принять решение за один экран**.

- 
**Given**  API вернул объект с названием и координатами.  
**When**  отображается карточка.  
**Then**  название и координаты (широта и долгота или одна строка координат) видны в первой видимой области карточки до первой прокрутки содержимого.

- 
**Given**  API вернул описание и медиа.  
**When**  пользователь смотрит карточку.  
**Then**  описание и превью/ссылки на медиа отображаются в соответствии с данными API; при отсутствии полей показывается аккуратное пустое состояние без «битых» блоков.

## US-E7

**История:** Как **деловой путешественник**, я хочу **видеть состояние загрузки при запросах к данным карт**, чтобы **понимать, что приложение работает, а не зависло**.

- 
**Given**  выполняется запрос каталога или построения маршрута.  
**When**  ответ API ещё не получен.  
**Then**  отображается индикатор загрузки (анимированный индикатор или скелетон списка) в области, где появится контент.

- 
**Given**  запрос завершён.  
**When**  данные получены или зафиксирована ошибка.  
**Then**  индикатор загрузки скрывается и показывается контент или сообщение об ошибке.

---

## US-E8

**История:** Как **деловой путешественник**, я хочу **получать ясное сообщение при сетевой ошибке или ограничении API**, чтобы **не терять время на угадывание причины**.

- 
**Given**  устройство не в сети или запрос завершился по таймауту.  
**When**  пользователь запускает загрузку каталога или построение маршрута.  
**Then**  отображается текст о проблеме сети и кнопка «Повторить», по нажатию повторяется тот же запрос.

- 
**Given**  API карт вернул код ограничения частоты запросов (например HTTP 429) или сообщение о превышении квоты.  
**When**  пользователь выполняет запрос к данным карт.  
**Then**  отображается понятное сообщение о временной недоступности сервиса без кодов ошибок в тексте для пользователя; кнопка «Повторить» доступна и повторяет запрос после паузы пользователя.
