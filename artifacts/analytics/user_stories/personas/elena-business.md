# Елена — деловой путешественник

Краткий контекст: командировки, мало времени; нужен **минимум шагов**: область → список или маршрут → **внешние карты**; интерфейс **сдержанный**, без лишнего визуального шума.

---

## Сценарий 1: «Счастливый путь» — короткая прогулка после переговоров

**User Journey:** После встреч Елена открывает приложение, **быстро находит город** через поиск, при необходимости один раз просматривает **короткий список** мест рядом с отелем, переходит к **«Собрать маршрут»**, вводит небольшой ориентир (**например, 5 км**), подтверждает старт от **геолокации** или точки у отеля, получает до трёх вариантов с **1…N** и длиной, выбирает компактный вариант («короче» или «около») и **открывает маршрут или точку во внешнем приложении карт** для прогулки вечером.

**User Stories:**

| ID | Формулировка |
|----|----------------|
| US-E1 | Как **деловой путешественник**, я хочу **задать город поиском в один-два шага**, чтобы **не тратить время на сложный выбор области**. |
| US-E2 | Как **деловой путешественник**, я хочу **собрать кольцевой маршрут по ориентиру в км со стартом от геолокации или выбранной точки**, чтобы **вписать прогулку в свободный слот**. |
| US-E3 | Как **деловой путешественник**, я хочу **видеть нумерацию порядка посещения и итоговую длину варианта**, чтобы **быстро сориентироваться без лишних экранов**. |
| US-E4 | Как **деловой путешественник**, я хочу **открыть маршрут или точку во внешнем приложении карт**, чтобы **идти привычным навигатором** (пошаговая навигация в приложении вне MVP). |

## Сценарий 2: Нормальный — только одна точка и карты, без полного маршрута

**User Journey:** У Елены **15 минут** между делами. Она задаёт город, открывает **одну карточку** (рекомендованное место рядом), сразу жмёт **открыть во внешних картах**, не запуская сценарий «Собрать маршрут».

**User Stories:**

| ID | Формулировка |
|----|----------------|
| US-E5 | Как **деловой путешественник**, я хочу **получить к карточке объекта быстрый путь «открыть в картах»**, чтобы **сразу выйти к навигации без сборки маршрута**. |
| US-E6 | Как **деловой путешественник**, я хочу **видеть в карточке достаточно данных из API (название, координаты, описание при наличии)**, чтобы **принять решение за один экран**. |

## Сценарий 3: Нормальный — ожидание ответа API и ясность при сбоях

**User Journey:** Слабый Wi‑Fi в отеле: при загрузке каталога и построении маршрута Елена видит **индикацию загрузки**; при **ошибке сети или лимите API** — **короткое понятное сообщение** с возможностью повторить запрос позже, без «немого» экрана.

**User Stories:**

| ID | Формулировка |
|----|----------------|
| US-E7 | Как **деловой путешественник**, я хочу **видеть состояние загрузки при запросах к данным карт**, чтобы **понимать, что приложение работает, а не зависло**. |
| US-E8 | Как **деловой путешественник**, я хочу **получать ясное сообщение при сетевой ошибке или ограничении API**, чтобы **не терять время на угадывание причины**. |
