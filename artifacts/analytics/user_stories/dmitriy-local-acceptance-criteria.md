# Критерии приёмки: Дмитрий — местный житель (US-D1 … US-D9)

Каждый критерий оформлен так:

- **Given** — начальное условие  
- **When** — действие пользователя  
- **Then** — проверяемый результат  

---

## US-D1

**История:** Как **местный житель**, я хочу **выбрать город или район без туристического «поиска с нуля»**, чтобы **быстро перейти к знакомой местности**.

- 
**Given**  открыт экран выбора локации.  
**When**  пользователь вводит название своего города и выбирает подходящую подсказку из результатов поиска.  
**Then**  область поиска устанавливается по выбранному элементу и загружается каталог для этой области.

- 
**Given**  пользователь ошибся в названии.  
**When**  он выбирает другой результат поиска.  
**Then**  каталог обновляется под новую область без сохранения предыдущей выборки как текущей.

---

## US-D2

**История:** Как **местный житель**, я хочу **фильтровать достопримечательности по категориям**, чтобы **подобрать места для прогулки с детьми**.

- 
**Given**  API возвращает категории для объектов региона.  
**When**  пользователь применяет фильтр по категории, подходящей для семейной прогулки.  
**Then**  список содержит только объекты с этой категорией (или пустой список с текстом, что под фильтром ничего нет).

- 
**Given**  в интерфейсе доступен выбор нескольких категорий.  
**When**  пользователь отмечает несколько категорий и применяет фильтр.  
**Then**  в списке остаются объекты, принадлежащие хотя бы одной из выбранных категорий (логика объединения «ИЛИ»).

- 
**Given**  в интерфейсе доступен только один активный фильтр категории.  
**When**  пользователь выбирает другую категорию.  
**Then**  список полностью заменяется объектами только этой категории.

---

## US-D3

**История:** Как **местный житель**, я хочу **указать старт маршрута точкой на карте или геолокацией**, чтобы **начать кольцо от реального места выхода из дома**.

- 
**Given**  открыт экран подтверждения старта.  
**When**  пользователь выбирает «Точка на карте» и ставит маркер в допустимой области.  
**Then**  координаты маркера используются как старт кольцевого маршрута при расчёте.

- 
**Given**  включена геолокация.  
**When**  пользователь выбирает старт «Текущее местоположение».  
**Then**  используются актуальные координаты устройства как старт; при недоступности GPS показывается сообщение и предлагается выбор точки на карте.

---

## US-D4

**История:** Как **местный житель**, я хочу **получить кольцевой маршрут с явной суммарной длиной и порядком остановок**, чтобы **спланировать нагрузку для семьи**.

- 
**Given**  построение маршрута успешно.  
**When**  пользователь просматривает любой вариант.  
**Then**  отображается суммарная длина в километрах или метрах и нумерация остановок 1…N; последняя остановка совпадает с первой (возврат к старту), суммарная длина включает все отрезки между остановками по порядку и замыкающий отрезок к старту.

- 
**Given**  отображается несколько вариантов.  
**When**  пользователь сравнивает карточки вариантов.  
**Then**  у каждого варианта своя итоговая длина, видимая без перехода в скрытые детали.

---

## US-D5

**История:** Как **местный житель**, я хочу **сохранить понравившийся маршрут в избранное**, чтобы **повторить прогулку или показать маршрут знакомым**.

- 
**Given**  пользователь на экране выбранного варианта маршрута.  
**When**  он сохраняет маршрут в избранное.  
**Then**  в локальном хранилище сохраняются суммарная длина, порядок остановок и координаты каждой остановки; запись появляется в списке избранного.

- 
**Given**  маршрут ранее сохранён в избранное.  
**When**  пользователь снова открывает раздел избранного на том же устройстве.  
**Then**  отображаются те же длина, порядок точек и координаты, что были сохранены.

---

## US-D6

**История:** Как **местный житель**, я хочу **быстро находить ранее сохранённые маршруты и объекты**, чтобы **не собирать маршрут заново**.

- 
**Given**  в избранном есть хотя бы один маршрут или объект.  
**When**  пользователь открывает раздел избранного.  
**Then**  отображается список с различением типа «маршрут» / «объект» и возможностью открыть элемент за один шаг от списка.

- 
**Given**  избранное пусто.  
**When**  пользователь открывает раздел.  
**Then**  отображается пустое состояние с подсказкой, как добавить маршрут или объект.

---

## US-D7

**История:** Как **местный житель**, я хочу **открыть точку маршрута во внешнем приложении карт**, чтобы **дойти привычным навигатором**.

- 
**Given**  пользователь просматривает маршрут или карточку точки из маршрута.  
**When**  он выбирает открытие во внешних картах для конкретной остановки.  
**Then**  широта и долгота этой остановки передаются во внешнее приложение карт через механизм ОС (Intent с `geo:` на Android, открытие карт по координатам на iOS).

- 
**Given**  координаты точки отсутствуют или некорректны.  
**When**  пользователь пытается открыть карты.  
**Then**  действие не выполняется и показывается сообщение об ошибке данных.

---

## US-D8

**История:** Как **местный житель**, я хочу **видеть явное сообщение, если нельзя собрать маршрут из‑за малого числа точек**, чтобы **скорректировать фильтр или зону поиска**.

- 
**Given**  в текущей выборке для маршрута доступно меньше трёх достопримечательностей.  
**When**  пользователь запускает построение маршрута.  
**Then**  успешный список вариантов не отображается; показывается сообщение о том, что нужно не менее трёх точек, и рекомендации расширить область или снять фильтр.

- 
**Given**  ранее отображалось сообщение о недостатке точек.  
**When**  пользователь расширяет область или снимает фильтр и снова запускает построение.  
**Then**  при наличии не менее трёх точек в выборке отображается экран с вариантами маршрута; при иной ошибке (сеть, API) показывается только сообщение этой попытки без смешения с предыдущим успехом.

---

## US-D9

**История:** Как **местный житель**, я хочу **снова запустить построение маршрута после изменения условий**, чтобы **получить рабочий вариант без путаницы**.

- 
**Given**  предыдущая попытка завершилась сообщением об ошибке или недостатке данных.  
**When**  пользователь изменил длину, фильтр, область или старт и снова нажал «Построить».  
**Then**  выполняется новый запрос к расчёту; на экране одновременно не отображаются карточки успешного маршрута от старой попытки и сообщение об ошибке новой (показывается актуальное состояние последней попытки).

- 
**Given**  новый расчёт успешен.  
**When**  пользователь смотрит экран результатов.  
**Then**  отображаются только данные актуальной попытки.
