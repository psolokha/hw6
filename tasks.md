# Домашнее задание

## Настройка CI/CD и интеграция сервисов

### Цель

- настроить автоматические пайплайны CI/CD для сборки и деплоя;
- провести аудит безопасности проекта с помощью AI-инструментов;
- интегрировать дополнительные сервисы (OAuth2, аналитика, платежи).

---

## Описание / Пошаговая инструкция выполнения домашнего задания

### Шаг 1. Настройка CI/CD пайплайна

- выбрать платформу для CI/CD (GitHub Actions, GitLab CI или другая);
- создать конфигурационный файл пайплайна:
  - использовать AI-агента для генерации базового пайплайна;
  - настроить этапы: установка зависимостей, сборка, тестирование, деплой;
  - добавить проверки качества кода (линтинг, форматирование);
- настроить автоматический деплой:
  - при пуше в main/master ветку;
  - на хостинг (Vercel, Netlify, Railway и т.д.);
- протестировать пайплайн на тестовом коммите.

### Шаг 2. Аудит безопасности

- провести аудит безопасности проекта:
  - проверить зависимости на уязвимости (npm audit, safety и т.д.);
  - проверить код на проблемы безопасности (OWASP Top 10);
  - использовать AI-инструменты для анализа кода;
- исправить найденные уязвимости:
  - обновить уязвимые зависимости;
  - исправить проблемы в коде;
  - добавить защиту от распространённых атак (XSS, CSRF, SQL injection);
- задокументировать найденные проблемы и исправления.

### Шаг 3. Интеграция OAuth2

- выбрать провайдера OAuth2 (Google, Yandex ID или оба);
- реализовать авторизацию через OAuth2:
  - настроить приложение в консоли провайдера;
  - получить Client ID и Client Secret;
  - реализовать flow авторизации на Backend;
  - обновить Frontend для использования OAuth2;
- протестировать авторизацию:
  - проверить успешный вход;
  - проверить обработку ошибок;
  - проверить получение данных пользователя.

### Шаг 4. Интеграция аналитики

- выбрать сервис аналитики (Яндекс.Метрика, Google Analytics или другой);
- интегрировать аналитику в приложение:
  - добавить код отслеживания на Frontend;
  - настроить события для отслеживания;
  - протестировать отправку данных;
- использовать AI для помощи в настройке событий.

### Шаг 5. Интеграция платежного сервиса (опционально)

- если требуется, интегрировать платежный сервис:
  - выбрать сервис (ЮKassa, Stripe или другой);
  - настроить обработку платежей на Backend;
  - создать интерфейс оплаты на Frontend;
  - настроить webhook для обработки уведомлений;
- протестировать платежи в тестовом режиме.

### Шаг 6. Настройка мониторинга

- настроить мониторинг приложения:
  - использовать встроенные инструменты хостинга;
  - или настроить внешний сервис (UptimeRobot, Pingdom и т.д.);
- настроить Health Check endpoints:
  - создать endpoint для проверки состояния приложения;
  - проверить доступность БД и внешних сервисов;
- настроить алерты при проблемах.

### Шаг 7. Настройка логирования

- улучшить систему логирования:
  - структурировать логи (JSON формат);
  - добавить уровни логирования (info, warning, error);
  - настроить централизованное хранение логов (если возможно);
- использовать AI для анализа логов:
  - создать промпты для поиска проблем в логах;
  - протестировать анализ типичных ошибок.

### Шаг 8. Тестирование и оптимизация

- протестировать все интеграции:
  - проверить работу OAuth2;
  - проверить отправку данных в аналитику;
  - проверить работу платежей (если интегрированы);
  - проверить работу CI/CD пайплайна;
- оптимизировать производительность:
  - использовать AI для предложений по оптимизации;
  - применить рекомендации.

### Шаг 9. Оформление результатов

- подготовить документацию:
  - описание настроек CI/CD;
  - инструкции по интеграции сервисов;
  - отчёт по аудиту безопасности;
  - описание мониторинга и логирования;
- обновить README в репозитории.

## Критерии оценки

**Система оценки:** статусы «Принято» / «Возвращено на доработку».

### Критерии для статуса «Принято»

#### Полнота выполнения (обязательно)

- выполнены все обязательные шаги (1–4, 6–7);
- предоставлены все требуемые артефакты.

#### CI/CD пайплайн (обязательно)

- пайплайн настроен и работает;
- автоматически собирает и деплоит приложение;
- включает проверки качества кода.

#### Интеграции (обязательно)

- интегрировано минимум 2 сервиса из списка:
  - OAuth2 авторизация;
  - Аналитика;
  - Платежи (опционально);
- интеграции работают корректно;
- нет критических ошибок.

#### Аудит безопасности (обязательно)

- проведён аудит проекта;
- найдены и исправлены уязвимости;
- отчёт по безопасности подготовлен.

#### Мониторинг и логирование (обязательно)

- настроен мониторинг приложения;
- настроено логирование;
- Health Check endpoints работают.

#### Качество кода и документации (обязательно)

- код качественный и структурированный;
- документация полная и понятная;
- инструкции работают.

#### Использование AI (обязательно)

- использован AI для генерации CI/CD конфигурации;
- использован AI для аудита безопасности;
- использован AI для анализа логов;
- процесс задокументирован.

### Основания для возврата на доработку

- CI/CD пайплайн не работает;
- не интегрировано минимум 2 сервиса;
- не проведён аудит безопасности;
- не настроен мониторинг;
- нет документации или она неполная;
- не продемонстрировано использование AI.
